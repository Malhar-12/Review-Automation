# ReviewPulse App

This is the React MVP for ReviewPulse, a review and enquiry automation workspace for clinics, hospitals, and specialty practices.

## What is included

- Vite + React setup
- interactive dashboard, reviews, patients, campaigns, enquiries, and settings screens
- browser persistence for offline demo mode
- Supabase client wiring with graceful local fallback
- Supabase email/password auth gate for the live dashboard
- SQL schema file for the first database setup

## Run locally

1. Install dependencies with `npm install`
2. Copy `.env.example` to `.env.local` if needed
3. Start the app with `npm run dev`
4. Open the local URL shown by Vite

## Supabase setup

1. Open your Supabase project
2. Go to `SQL Editor`
3. Run the SQL in [supabase-schema.sql](./supabase-schema.sql)
4. Make sure `.env.local` contains:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`

The SQL file is now a tenant-safe migration, so you can rerun it on an existing project to move from shared demo tables to per-user clinic data. When the tables and policies are ready, the app will sync there automatically. If they do not exist yet, the app stays usable in local demo mode.

## Authentication setup

1. In Supabase, open `Authentication`
2. Enable the `Email` provider
3. Create a user from the app sign-up screen or from the Supabase dashboard
4. Sign in to access the protected dashboard

## Notes

- Auth is connected with Supabase email/password
- Each signed-in user now gets their own Supabase-backed practice workspace
- Review drafts are still local, not generated by OpenAI yet
- WhatsApp automation is still a planned next step
